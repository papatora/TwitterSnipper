# Twitter Sniper Rules

## Purpose
This file defines the discovery, thesis, keyword expansion, and reject taxonomy for a Twitter sniper workflow focused on finding high-signal accounts and projects early. The goal is to prioritize builders, infra, primitives, and emerging narratives while filtering out farming, low-signal KOL behavior, generic community accounts, and noisy crypto CT profiles.

---

## Discovery Sources
Keep discovery fixed to:
- following
- search
- xreach

Do not modify the seed list. Keep the seed list intact and improve quality through:
- better keyword thesis
- better chain-specific combinations
- stronger reject taxonomy
- better early-signal phrasing

---

## Core Thesis Focus
Primary focus areas:
1. ARC
2. Tempo
3. TAO / Bittensor / subnets
4. AI, but only web3 / onchain / decentralized AI
5. Solana only as secondary thesis, mainly through quantum or inscription angles
6. Cross-chain inscription narratives tied to the active thesis chains
7. Cross-chain quantum narratives tied to the active thesis chains
8. Early or first-of-category projects across each thesis area

The thesis should stay tightly connected to chain names or project names so search remains easy and targeted.

---

## Core Query Structure
Use combinations in this format:

[chain/project] + [narrative] + [action/state]

Examples:
- arc + quantum + launch
- tempo + payments + infra
- tao + subnet + live
- inscription + arc + experiment
- ai + onchain + agent
- quantum + arc + primitive

Avoid relying only on single broad keywords.

Weak examples:
- ai crypto
- arc nft
- solana project

Better examples:
- arc quantum build
- tempo payments infra
- tao subnet launch
- inscription arc experiment
- ai agent onchain tao

## Practical Search Guidance
When building search queries:
- keep chain names or project names in the query
- combine narrative + chain + early-signal wording
- prefer infra and builder language over community language
- do not over-rely on broad nft or meme words without chain context
- do not search AI without clear web3 context
- do not search Solana broadly outside inscription or quantum angle
- use first / genesis / native / primitive / beta variations to surface early projects
