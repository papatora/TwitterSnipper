OVERVIEW
--------
Sistem ini terdiri dari 2 script utama:

  1. nft_sniper.py   — Discovery bot (MAIN)
                       Monitor recent following CT seed accounts → classify →
                       NFT CT → main Notion | CT only → reject Notion | lain → skip

  2. run_follow_v2.py — Follow bot (terpisah, jalankan manual)
                        Follow 1000 seed accounts menggunakan 26 sessions
                        Rate limit: 14 follow / 15 menit per auth_token
                        Butuh ~62 jam untuk selesai semua

Script pendukung:
  frontrun_check.py        — Check smart account frontrun untuk CT list
  test_sniper_ratelimit.py — Probe rate limit Twitter (sudah dijalankan, jangan diulang)
  notion_reset.py          — Archive semua entry Notion (gunakan hati-hati)

================================================================================
  COOKIE SESSION FORMAT
================================================================================

Setiap file cookie = satu baris per akun Twitter, format:
  auth_token=<redacted>; ct0=<redacted>; twid=u%3DZZZZ

Field wajib: auth_token, ct0
Field opsional: twid (untuk identifikasi uid di log)

Cara dapat cookie:
  1. Login Twitter di browser
  2. F12 → Application → Cookies → twitter.com
  3. Copy nilai auth_token dan ct0
  4. Tulis: auth_token=<redacted>; ct0=<redacted>

Cookie expired → script otomatis skip (status [DEAD])
Session valid check: GraphQL UserByScreenName (bukan verify_credentials yg deprecated)

FLOW KERJA (per cycle):
  a. Load sessions + validasi ALIVE/DEAD
  b. Load dedup sets: sniper_rejected.txt + sniper_seen.txt
  c. Load main Notion cache (existing handles)
  d. Sort seeds berdasarkan last_yield (adaptive rotation)
  e. Untuk setiap CT seed:
       - Hitung fetch_count (base atau bonus jika seed aktif)
       - Ambil N recent following → 1 request saja
       - Diff dengan state sebelumnya → siapa yang baru di-follow?
  f. Per akun baru, cek gate berurutan:
       Gate 1: sudah di Notion / sniper_seen.txt    → SKIP
       Gate 2: pernah di-reject / sniper_rejected.txt → SKIP
       Gate 3: classify_account() → routing 3 arah (lihat bawah)
  g. Update sniper_state.json, sniper_rejected.txt, sniper_seen.txt

Setiap akun baru yang ditemukan diklasifikasikan:

================================================================================
  ADAPTIVE ROTATION
================================================================================

  - Seeds diurutkan berdasarkan last_yield (paling produktif duluan tiap cycle)
  - Seed AKTIF  : yield >= 50% dari fetch_count di cycle sebelumnya
                  → dapat bonus fetch (FETCH_PER_SEED * FETCH_BONUS_MULTIPLIER = 30)
  - Seed STALE  : yield < 50%
                  → tetap dicek, tapi dengan jatah minimum (FETCH_PER_SEED = 15)
  - BASELINE    : seed pertama kali dicek → pakai base fetch, yield = 0
  - Semua seed selalu dicek tiap cycle (tidak ada yang di-skip)
  - State per seed di sniper_state.json: known_ids, last_yield, last_fetched

  Log per seed:
    @supraEVM            → 7/15 new  yield=47%  [STALE]
    @0itsali0            → 8/30 new  yield=27%  [ACTIVE]
    @waleswoosh          → 0/15 baseline         [BASELINE]


================================================================================
  VARIABLE PENTING (di script, bisa diubah)
================================================================================

  FETCH_PER_SEED         = 15    # berapa recent following per seed (base)
  FETCH_BONUS_MULTIPLIER = 2     # bonus multiplier untuk seed aktif (15 * 2 = 30)
  HIGH_YIELD_THRESHOLD   = 0.5   # ratio new/fetched >= ini → seed dianggap aktif
  INTERVAL_MIN           = 20    # interval loop default (menit)
  FOLLOWERS_CAP          = 1000  # max followers untuk masuk list (early accounts)
  FRONTRUN_SLEEP         = 0.5   # jeda antar frontrun API call (detik)
  NOTION_SLEEP           = 0.35  # jeda antar Notion upsert (detik)

  TIER SCORING (Alpha Score)

  Smart account (frontrun verified wallet/position) : +40 pts
  Followers  100–1k                                 : +10
             < 100                                  : +5
  NFT keyword di bio                                : +10
  Account age >= 3 tahun                            : +20
  Account age >= 1 tahun                            : +10

  Tier A  : score >= 70  (hijau)
  Tier B  : score >= 40  (kuning)
  Tier C  : score < 40   (abu-abu)


================================================================================
  NOTION DB — "NFT Early List" (main)
================================================================================

Field yang diisi otomatis oleh nft_sniper.py:

  Akun            : nama Twitter
  Link            : https://x.com/handle
  Follower        : jumlah follower saat crawl
  Bio             : bio Twitter
  Tier            : A / B / C (auto-scored)
  Smart account   : checkbox (dari frontrun)
  Category        : NFT (fixed)
  Source Seed     : following:@supraEVM (dari mana akun ini ditemukan)
  Account Created : tanggal buat akun Twitter (bukan tanggal ditambah ke Notion)
  Added date      : tanggal pertama kali masuk Notion
  Last Checked    : tanggal terakhir di-update
  Alpha Score     : angka raw score

Update behavior:
  - Akun baru     → CREATE entry lengkap
  - Akun existing → UPDATE: Last Checked, Tier, Smart account, Follower, Alpha Score
                    (Added date dan Source Seed tidak berubah)

================================================================================
  TROUBLESHOOTING
================================================================================

Semua session DEAD:
  → Cookie expired. Update cookie files dengan auth_token + ct0 baru dari browser.
  → Pastikan format benar: auth_token=<redacted>; ct0=<redacted> (pisah titik koma)

HTTP 403 pada follow (run_follow_v2.py):
  → Akun Twitter restricted/baru. Normal untuk beberapa akun dari new auth.txt

friends/list HTTP 200 tapi users kosong:
  → Akun seed mungkin protected/private

Notion upsert gagal HTTP 400:
  → Cek format property. Field baru belum ada di DB.
  → Jalankan notion_reset.py untuk setup ulang schema (HATI-HATI: archive semua entry)

Frontrun API gagal:
  → Token frontrun expired. Ambil token baru dari browser (cookie frontrun.pro)
  → Script tetap jalan, smart=False untuk akun yang gagal dicek

Script berhenti di tengah:
  → sniper_state.json menyimpan state. Jalankan lagi, lanjut otomatis.
  → Untuk mulai ulang dari awal: hapus sniper_state.json

Notion reject DB gagal:
  → Cek REJECT_NOTION_TOKEN di config. Token berbeda dari main Notion token.
